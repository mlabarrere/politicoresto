# Documentation PoliticoResto

Ce dossier contient la documentation canonique du projet.

## Canonique (a maintenir en priorite)

- `metier.md`: cadrage produit, parcours utilisateurs, regles metier visibles.
- `technique.md`: architecture applicative, responsabilites par couche, patterns de code.
- `front-back-contract.md`: contrat SQL consomme par le frontend et garde-fous de derive.
- `runbook-prod.md`: exploitation, diagnostics, incidents et checklists release.
- `testing-strategy.md`: pyramide de tests, parcours critiques, standards de non-regression.

## Documentation par sous-systeme

- `../frontend/README.md`: setup frontend, conventions d'implementation.
- `../frontend/docs/README.md`: details UI/Catalyst et audits UX/UI.
- `../supabase/docs/README.md`: documentation SQL, contrats data et validation locale.
